API Refuges des Pyrénées

Données normalisées des refuges, cabanes et abris du versant nord des Pyrénées et d'Andorre — en JSON et GeoJSON, librement utilisables avec attribution.

Points d'accès

GET/api/refuges.jsontous les lieux
GET/api/refuges.geojsonGeoJSON (SIG, Leaflet…)
GET/api/refuges-occitanie.jsonpar région
GET/api/refuges-andorre.jsonpar région
GET/api/refuges-nouvelle-aquitaine.jsonpar région
GET/api/refuges-cat-refuge.jsonrefuges gardés
GET/api/refuges-cat-libre.jsoncabanes ouvertes
GET/api/refuges-cat-cabane.jsoncabanes / abris
GET/api/refuges-cat-ruine.jsonruines

Format d'un lieu

ChampTypeDescription
idstringIdentifiant stable (nom + coordonnées)
nomstringNom du lieu
lat, lonnumberCoordonnées WGS84
altitudenumber | nullAltitude en mètres
regionstringAndorre, Occitanie ou Nouvelle-Aquitaine
placesnumber | nullCouchages estimés
eaustring | nullEau à proximité (déclaratif source)
boisstring | nullBois à proximité (déclaratif source)
type_numnumber | nullType source (1 à 7)
type_libellestring | nullLibellé du type source
categoriestringrefuge, libre, cabane ou ruine
descriptionstring | nullDescription texte (HTML retiré)
lienstring | nullURL externe éventuelle

Exemple

const rep = await fetch('https://TON-SITE/api/refuges.json');
const { meta, refuges } = await rep.json();

console.log(meta.nombre, 'lieux —', meta.genere_le);
const gardes = refuges.filter(r => r.categorie === 'refuge');
# En ligne de commande
curl -s https://TON-SITE/api/refuges-cat-refuge.json | jq '.refuges[0]'

Fraîcheur des données

Les fichiers sont régénérés à chaque déploiement du site (champ meta.genere_le). Pour les mettre à jour : node outils/generer-api.mjs, ou ouvrir outils/generer-api.html dans le navigateur puis copier les fichiers téléchargés dans api/.

Attribution requise. Les données proviennent de pyrenees-refuges.com. Toute réutilisation doit conserver cette attribution. Les informations (eau, bois, état…) sont déclaratives et non vérifiées sur le terrain : à confirmer avant de partir.
